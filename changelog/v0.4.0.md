# v0.4.0 — 2026-04-27

_Since [v0.3.0] (2026-04-24)._

## Features

- **COLLADA (.dae) import** — `.dae` files now import alongside GLTF, sharing the same parts/colors/material-mapping pipeline. Stored `rawSource` is the XML string for the 3D viewer (#12).
- **Mobile-friendly project workspace** — responsive `ProjectTopBar` and `ProjectWorkspaceNav` for small screens, touch-tuned pan/zoom on the layout preview, and a viewport meta tag so the app renders correctly on phones.
- **New project dialog** — extracted project creation into a dedicated `NewProjectDialog` component.

## Code Quality

- **Component reorganisation** — components moved into domain folders (`bom/`, `layout/`, `project/`, `project/tabs/`, `project/workspace/`, `app/`), with a new `ProjectWorkspace` shell composing nav, tabs, and modals.
- **`useProjects` decomposition** — split the monolith composable into `useActiveProject`, `useProjectCollection`, `useProjectModels`, plus shared `state.ts` / `types.ts` and a new `useProjectNavigation` helper.
- **Shared model-parse pipeline** — extracted `resolveModelScene`, `groupPartInfos`, and `modelTypes` so GLTF and COLLADA imports run through the same derive pipeline at import time.
- **Computation worker simplification** — dropped the route-level `computation-guard` middleware in favour of in-composable guards; trimmed `useComputationWorker` and `useBoardLayoutsQuery` lifecycle handling.
- **Test suite expansion** — added Vue component tests (`ManualPartRow`, `RulerToggle`, `ScaleController`, `TabListItem`, `GrainLockConfirmModal`, `ProjectWorkspaceNav`) and broader coverage on `useManualParts`, `useModelHydration`, `parseCollada`, and `parseStock`.

## Infrastructure

- **Vitest migration** — replaced `bun test` with Vitest + `@nuxt/test-utils`. Pure-logic and IDB tests run under `happy-dom`; component tests opt into the Nuxt environment per file with `// @vitest-environment nuxt`.
- **`write-tests` skill** — internal skill capturing project test conventions for future test authoring.
