# v0.8.0 — 2026-05-06

_Since [v0.7.1] (2026-05-05)._

A short follow-up to the v0.7 build-doc rework: a view/edit split on the Build tab, drag-and-drop / paste image insertion in the editor, and Objects-panel polish so renames surface and large assemblies are scannable on load.

## Features

- **Build tab view/edit toggle** — populated build docs now open in a rendered view with an Edit button; empty docs land directly in the editor. Done flushes pending writes and switches back to view.
- **Image drag-drop & paste in the build doc** — dropping or pasting an image into the Tiptap editor now inserts it via the existing upload path (parallel uploads, project-switch race guard) instead of navigating the page. Adds `@tiptap/extension-typography` for em-dash, smart-quote, and fraction input rules in editorial prose.
- **Objects panel reflects part renames** — the panel was reading names off the cached `ObjectGraph`, so user renames stored in `partOverrides` never surfaced. Hydrated part names now overlay the graph names.
- **Objects default to collapsed** — every part group loads collapsed (assemblies with 100+ Objects were unscannable fully expanded); selecting an Object still auto-expands its group via the existing scroll-into-view watcher.

## Reliability

- **Image upload policy** — PNG/JPEG/WebP and a 15 MB cap are now enforced inside `useDocAssets`, so every upload path (toolbar picker, drop, paste) hits the same boundary.
- **`useBuildDoc` project-switch race** — the watcher used to await a flush and an IDB read before clearing live refs, so the previous project's doc rendered next to the new project's identity during the gap. The watcher now snapshots any pending write, resets visible state synchronously, fires the snapshot in parallel, and a stale-load guard drops a slow read whose project is no longer active.
- **Scene-embed thumbnails after import** — a freshly-imported buildDoc lands before `activeProject` rehydrates, so the scene picker's `eligibleModels`-driven fetch missed the embed's `modelId` and the thumbnail stayed blank until reload. The picker now also tracks the embed's own `modelId`, with a generation counter to discard stale async results.

## Bug Fixes

- **Scene embed dark overlay** — removed the unintended dark overlay on scene thumbnails in build-doc embeds.
- **BOM heading spacing** — small layout tidy on the BOM tab heading.

## Infrastructure

- **`pages-deploy.yml` cleanup** — trimmed unused workflow steps.
