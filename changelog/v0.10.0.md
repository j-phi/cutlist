# v0.10.0 ‚Äî 2026-05-08

_Since [v0.9.0] (2026-05-06)._

A layout-engine rebuild release. The packing engine gains a new two-stage Tidy packer, multi-board lookback across guillotine and tight packers, and a per-(material, thickness) algorithm picker on the Layout tab. The search space shrinks from 17 passes to 5 with no fixture regressions. Plus Sentry error monitoring with an in-app feedback button, a homepage redesign, and viewer load polish.

## Features

- **Tidy two-stage guillotine packer** ‚Äî New `TidyPacker` partitions each bin into rip-axis strips whose widths are set by the first part dropped in (so similar widths cluster into the same column), then stacks via Best-Fit Decreasing within each strip. Within-strip side residuals stay reusable, and multi-board lookback is supported.
- **Per-thickness algorithm picker** ‚Äî The Layout tab now shows a dropdown per (material, thickness) group that pins Auto / Tidy / Compact / Tight (CNC). Schema bumped to v2: project-level `optimize: 'Auto' | 'CNC'` becomes `defaultAlgorithm: Algorithm` plus `StockMatrix.thicknessAlgorithms` overrides, with parallel migrations on the Dexie and `.cutlist.gz` import paths.
- **Multi-board lookback in guillotine and tight packers** ‚Äî Both packers now iterate parts one at a time and try every already-opened board before opening a new sheet. Fixes the "sparse last board" anomaly where small parts stranded on a fresh sheet despite obvious gaps on earlier ones. Cabinet-mixed fixture: 6 ‚Üí 5 boards, 29.4% ‚Üí 15.2% waste. Shelves-bulk: 5 ‚Üí 4 boards, 32.8% ‚Üí 16.0% waste.
- **Fit-to-model viewer button** ‚Äî New button beside the ViewCube; `ViewerCore.fit` frames currently-visible parts (falling back to full scene bounds when none are visible). Floor still tracks the full scene bounds.
- **In-app feedback / "Help" button** ‚Äî Header gets a Help button that opens the Sentry feedback form so issues can be reported without leaving the app.
- **Homepage redesign** ‚Äî Updated landing-page hero, illustrations, and copy.

## Performance

- **Smoother model viewer load** ‚Äî `useFocusedModelLoader` now reloads scenes before `viewer.loadModel` and fires `jumpToScene` synchronously after, so the renderer paints once at the active scene's pose instead of snapping from the bounds-fit pose to the saved camera.

## Reliability

- **Sentry error monitoring** ‚Äî Sentry is now wired into the client config and captures project import / export, asset uploads, PDF export, and model loading errors. Per-environment (development / production) configuration with structured events for project CRUD operations.
- **Layout-engine review follow-ups** ‚Äî Per-group algorithm resolution scans for the first defined override across all stocks in the group rather than reading `group.stock[0]` directly, so duplicate `StockMatrix` rows can't let an undefined entry beat a defined override. `LayoutList.setOverride` writes to every matching row and `preferenceFor` reads from any matching row, keeping the picker in sync with what the engine runs. `placeAllPartsSingleBoard` (unreachable now that every active packer exposes lookback hooks) replaced with an explicit runtime guard.

## Bug Fixes

- **Project list ordering** ‚Äî `getProjectList` no longer reverses the `orderBy('updatedAt')` query; the consumer expected ascending order, so the sidebar now lists projects as intended.

## Code Quality

- **Trimmed packer set and search passes** ‚Äî Deleted `ShelfPacker` (degenerate row packer) and `StripPacker` (visibly worse layouts than guillotine + lookback on every fixture). Dropped guillotine BAF/BLSF variants (BSSF wins everywhere) and collapsed three CNC random seeds to one. Default search passes: 17 ‚Üí 5, no benchmark regressions.
- **`GuillotinePacker` renamed to `CompactPacker`** ‚Äî Same n-stage free-rect guillotine algorithm; the new name reads more honestly now that Tidy is also guillotine.
- **Cut-complexity scoring is stage-aware** ‚Äî Per-board the rip axis (the dimension with fewer unique edges) is weighted 10√ó the cross axis, so tidy column-aligned layouts win ties over compact zigzag. New `wasteConcentration` tiebreaker (sum of squared per-board waste areas) means total-waste ties now favour layouts that consolidate parts onto earlier boards.
