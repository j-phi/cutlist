# v0.11.0 — 2026-05-11

_Since [v0.10.2] (2026-05-08)._

A dimensional-lumber release. Cutlist now packs 1D timber stock (2×4s, CLS, dowels) alongside sheet goods, the dimensions system canonicalises to millimetres at rest with a user-configurable display precision, and the Stock tab is rebuilt around a shared card that handles both sheet and timber materials with live dimension drafts.

## Features

- **Linear timber (1D) packing** (#18) — New `LinearPacker` (first-fit-decreasing) routes parts whose material is sold as sticks, not panels. Stock matrices gain a `kind: 'linear'` variant with a fixed cross-section and one or more stock lengths; the engine groups parts by (material, cross-section) and packs each group as 1D bins with kerf between cuts. Layout tab gains a stick view + per-material shopping list; PDF export covers both. Timber preset library ships with imperial (Pine 2×4 / 2×6, etc.) and metric CLS profiles.
- **Unified Stock tab** — Single flat list of stock entries replaces the old per-material grid. New `StockCard` handles sheet and timber with shared dimension-draft inputs that don't reformat mid-typing. Add via preset menu (filtered to the project's active unit) or custom sheet / timber. Duplicate-name collisions are flagged inline and auto-suffixed on add (trim + case-insensitive).
- **User-configurable display precision** — Per-project setting picks the rounding for displayed dimensions: 1/8" / 1/16" / 1/32" / 1/64" for inches, configurable step for mm. Storage stays raw; precision only affects what gets rendered and what edit-prefill snaps to.
- **Dimension inputs follow project unit** — Every dimension input (Stock, BOM manual parts, Settings) reads and writes in the project's `distanceUnit`. Imperial parses fractions, mixed numbers, feet+inches (`1' 6"`, `1ft 6in`), and unit glyphs.
- **Persistent BOM models-panel + sticky table headers** — Models-panel collapse state survives reload; BOM table headers stick to the top while scrolling long part lists.
- **Standardised dimension display order** — Length × Width × Thickness everywhere (BOM, hover, PDF, viewer labels).
- **Dolly toward cursor in 3D viewer** — Scroll-wheel zoom now dollies toward the cursor position instead of the scene origin, matching SketchUp / Fusion behaviour.

## Bug Fixes

- **New project dialogue** — Fixed a layout/copy regression in the create-project modal.
- **Stick view rendering** — Sticks now render at absolute pixel scale matching the sheet renderer (a 96″ stick and a 96″ sheet sit at correct relative size); previously all sticks rendered at the same width regardless of length.
- **Editable lengths in LinearStockInput** — Adding, editing, and removing length rows now emits the right updates.
- **BOM + hover blind to linear-routed parts** — BOM aggregation and hover info now resolve material/dimensions for parts placed on linear stock.
- **Sheet + linear layout sections separated** — Grid `auto-flow-column` keeps sheet and timber sections from interleaving on the Layout tab.
- **Trailing offcut overstates by one kerf** — `wasteEndM` now subtracts the kerf that separates the last cut from the offcut, so the shopping-list trim length matches what comes off the saw.
- **Stable Stock card identity on delete** — Removing a card no longer drags its successor's in-flight dimension drafts onto the wrong material (was index-keyed; now stable per-row ids).
- **Material name dedupe is whitespace + case-insensitive** — `"Pine"`, `"pine "`, and `"Pine "` now flag as duplicates and auto-suffix on add; previously identical-to-a-human names slipped through the exact-string check.

## Breaking Changes

- **Schema v3 — distances canonicalised to millimetres** — All distance fields (`bladeWidth`, `margin`, `Stock.sizes.*`, `Part.size.*`) are now stored in millimetres regardless of `distanceUnit`. Existing IndexedDB data and `.cutlist.gz` exports migrate automatically via the per-version v3 migration, which converts pre-v3 records from their declared `unit:` (or project `distanceUnit` fallback) and drops `unit:` from stock rows. Engine still works in meters internally; the display layer applies `distanceUnit` and `precision` to render.

## Reliability

- **Resilient stock parsing** — `parseStock` now drops malformed rows per-record instead of failing the whole list; the rest of the YAML still hydrates so a single bad row can't blank the Stock tab.
- **Dimension schema rejects 0, negatives, NaN, ∞** — `PositiveDimension` at the schema boundary catches values that previously propagated as silent bugs (a 0-width sheet "fitting" any 0-wide part) or blew up layout math (NaN / Infinity).
- **`parseDimension` clamped to a plausible maximum** — Pasted scientific notation (`1e308`) or unbounded typos are rejected at the parser rather than landing in storage.
- **v3 migration hardened against malformed input** — Migration runs inside a Dexie transaction; per-version transforms are now wrapped so unparseable rows drop cleanly instead of throwing and rolling back the upgrade (which would lock the user out).

## Code Quality

- **Stock and BoardLayout are `kind`-discriminated unions** — Same-shape `kind: 'sheet' | 'linear'` discriminator drives both the engine input (`StockMatrix`) and output (`BoardLayout`), and the intermediate `PotentialBoardLayout` — no runtime guard at serialize time, no `as` casts in app code.
- **Dimension formatters unified; 25.4 centralised** — Single `units.ts` module owns `MM_PER_IN`, `mmToM` / `mToMm`, `parseDimension`, `formatValue` / `formatDistance`, and the `Precision` type. Display-layer composables (`useDimensionInput`, `useDimensionDrafts`, `useFormatDistance`) bridge mm storage to the displayed string.
- **Per-version migration files** — `web/utils/projectImport/migrations/v3.ts` etc. each export a `RecordMigration`; `migrations/index.ts` is a version-agnostic registry. The Dexie `db.ts` upgrade blocks and the export-file `migrateExport()` both walk the same registry. Documented in CLAUDE.md.
- **Collapsed v4 migration into v3** — v3 was unshipped (no v3 users existed), so v4's `kind` discriminator folds into v3 as a single canonical migration step.
- **Drop hardwood lumber from imperial stock defaults** — Imperial new-project defaults seed sheet goods only; users add timber explicitly.
- **Inlined 7 single-caller composables** — Composables with one caller and no reactive state moved to the call site.
- **Trim ~2,300 LOC of tautological tests** — Vue template-forwarding emits, lifecycle no-ops, type-checker constant duplicates, object-spread idempotency cases, render-state cardinality smoke tests collapsed or dropped per the tautology check documented in CLAUDE.md. Shared component stubs in `web/test-utils/stubs.ts` replace per-file duplicates. Net coverage unchanged.
- **Per-thickness algorithm override moves to the Layout tab** — Removed from the new Stock tab; the field still flows through the schema and engine, edited via the Layout tab dropdown where users see board-by-board context.
