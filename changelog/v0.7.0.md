# v0.7.0 — 2026-05-05

_Since [v0.6.1] (2026-04-27)._

Two big rebuilds ship together: a ground-up rewrite of the 3D viewer (#13) introducing scenes, annotations, and gizmo-driven exploded views with a clean two-layer split between Three.js and Vue; and a Notion-style **Build doc** (#14) that replaces the step-list with a free-form rich-text page supporting image and 3D-scene embeds. The Model tab is restructured onto CSS Grid trays, making scenes a first-class part of build documentation: capture once in the Model tab, embed live in the doc.

## Features

### Viewer

- **Viewer rewrite** — replaced the legacy viewer with `ViewerCore` (plain TS) + `useThreeViewer` (thin Vue adapter). Three.js types no longer cross the boundary, leaving the door open to a worker or engine swap. Composed of single-responsibility modules (`Renderer`, `SceneGraph`, `CameraRig`, `InputRouter`, `BatchLoader`, `Floor`, etc.).
- **Camera rig + view cube** — orbit/pan/zoom with persisted zoom and up vector, plus an interactive view cube for snapping to standard angles.
- **Camera menu** — top-right dropdown for projection (perspective/orthographic) and floor visibility, alongside the orientation cube.
- **Objects panel** — browse and toggle visibility of every Object in the loaded model. Selecting an Object in the viewer scrolls its row into view, expanding its Part if collapsed.
- **Scenes system** — per-model scene timeline with camera pose, per-Object rigid offsets (translation + rotation), visibility set, and floor toggle. Scenes capture, jump, and tween between states with a single ease curve. A bottom scenes strip stuck to the canvas surfaces a per-card "Update" overlay on the active scene when viewer state has drifted from what was captured.
- **Transform gizmo** — `TransformControls`-driven gizmo for authoring exploded views, with an interaction lock that gates orbit/pick while dragging.
- **Annotations** — new framework supporting **callouts** (multi-line labels with snap-aware leaders) and **linear dimensions**. Annotations are anchored in Object-local space so they ride explode tweens and gizmo drags. Edit/remove flows share a unified two-step authoring FSM.
- **Top-center annotation toolbar** — gizmo Move/Rotate joins Callout/Dimension in a single pill toolbar; collapses to hint text in pick mode.
- **Marquee multi-select** — click-drag to select multiple objects in the viewer.
- **Visibility shortcut** — Delete/Backspace toggles visibility of the current selection.
- **Selection overlay** — selected objects render with a depth-test-free highlight so they remain visible through occluders.
- **Viewer status bar** — bottom edge shows a compact `#N Name  W × L × T` readout when a part is hovered or selected.
- **Crossfade on scene transitions** — body alpha tweens 0↔1 for objects whose visibility differs between scenes instead of hard-cutting at the midpoint; edge lines snap at the fade midpoint to mask the pop.
- **Model viewer CSS Grid layout** — sidebars and the scene strip are now grid trays that resize the canvas when toggled, so scene thumbnails capture exactly what the user sees. Grid `1fr` tracks give the sidebar a deterministic scrollable height that flex couldn't enforce.

### Build doc

- **Build doc editor** — replaces the step-list build instructions with a single free-form Tiptap page per project. Inline image and 3D-scene embeds are atom, draggable Tiptap nodes; scene blocks mount a read-only viewer on demand. Sticky toolbar with heading dropdown, B/I/list/link, and image/scene insert. Auto-saves on a 400ms debounce.

### BOM & Layout

- **BOM model preview** — read-only viewer embedded in the BOM tab.
- **BOM hover fan-out** — hovering a BOM row now highlights every Object that belongs to that Part, not just one.
- **Layout tab fit-to-screen on load** — `usePanZoom` now fits the cutlist content to the viewport on first mount instead of pinning to a fixed initial zoom.

## Performance

- **Proportional edge linewidth** — viewer edge width tracks the model's projected on-screen diameter each frame (with a low ceiling) instead of a constant pixel width, so edges no longer feel heavy on small models or wispy on large ones.
- **Higher-quality scene thumbnails** — captured at 1024×576 (16:9, matches the build-doc display) instead of 320×240 (4:3), with `LineMaterial.resolution` and proportional linewidth retargeted for the capture size. Output is JPEG q=0.92 to keep the per-scene IDB payload manageable.

## Reliability

- **Scene dirty tracking** — floor toggle and camera-mode changes now mark the active scene dirty; camera zoom + up are persisted alongside the pose.
- **Callout persistence across scenes** — fixed callout text disappearing after switching scenes.
- **Focused-model lifecycle** — fixed several issues around (re)loading the focused model and replaying its remembered active scene.
- **Embed block bleed** — removed stray padding/margin that bled past embed block boundaries in the build doc.
- **Sentry DSN** — corrected the misconfigured DSN so error reporting works in production.

## Code Quality

- **Outcome-based test rewrites** — converted the heaviest mock-laden suites to use real `THREE.*` math, real `EventBus`, and recording arrays instead of `vi.fn()` introspection: `GizmoController`, `MarqueeSelector`, `InputRouter`, `SnapDetector`, `useAnnotationAuthor`, `useTiptapEditor`, `useBomImport`, `BomTab`, `ProjectTopBar`.
- **Type unification** — consolidated viewer-internal and domain types; added a `ready` event so consumers can wait on viewer init deterministically.
- **Scene-state collapse** — folded `AppliedSceneState`, tween refs, and `applySceneState` into a smaller surface area.
- **ModelTab decomposition + ObjectGraph caching** — split the tab into focused composables; module-level cache keyed by modelId avoids re-running GLTFLoader on dropdown switches.
- **Selection axis** — collapsed model-viewer selection to a single `groupId` axis.
- **Module extraction** — pulled `BatchLoader` and `Floor` out of `ViewerCore`; extracted `BomSortableHeader` to dedupe BOM column headers; **`SceneViewer` extracted** as a dedicated read-only embed component (canvas + read-only annotations, no chrome) rather than a re-entrant ModelTab.
- **Component cleanup** — removed `GizmoModeToggle` and `MouseLegend` (folded into the new annotation toolbar).
- **Dead-branch sweep** — dropped unused `side: 'left' | 'right'` on `ViewerSidePanel`, `showGizmoControls` on `AnnotationToolbar`, and redundant predicates in `ModelTab` (`hasModelData`, the duplicate `enabledModels.length === 0` check in `emptyStateType`).
- **`useBuildDoc` module-scoped** — single watcher and a single `putBuildDoc` write path so multi-component callers don't multiply effects.
- **New test coverage** — added `useThreeViewer` smoke tests and tests for the export/import/urlSync flows.

## Infrastructure

- **CI parallelism** — split `lint`, `typecheck`, and `test` into parallel jobs for faster pipeline turnaround.

## Breaking Changes

- **Schema flatten** — IDB schema collapsed to a single v1 baseline (covering both the viewer-era and build-doc work). Existing databases from earlier dev builds need to be reset.
- **Object offsets are rigid transforms** — widened from translation-only to translation + rotation, changing the on-disk shape of scene records.
- **Scenes scoped to model** — scenes now belong to a specific `IdbModel` (previously project-scoped). The simultaneous multi-render plumbing has been removed; the Model tab renders one model at a time.
- **Build doc replaces buildSteps** — new `buildDocs` and `assets` tables; the old `buildSteps` table is gone. The build doc body is stored as Tiptap JSON (not rendered HTML), with embedded image and scene nodes serialising their referenced ids so they round-trip through export/import.
- **`.cutlist` import compatibility** — exports written before the build doc landed will fail with `LegacyExportError` and need to be re-exported from a current build.
