# v0.14.0 — 2026-05-13

_Since [v0.13.0] (2026-05-12)._

A stock-and-import release. Cutlist now suggests stock for you (cluster BOM cross-sections, match against timber presets, one-click "Add all"), accepts DAE / FBX / GLB files universally via Assimp WASM, and carries a material allowance on every linear stock so finished-dimension parts can ride rougher boards. Under the hood, the engine and storage move to integer micrometres — placement geometry is now exact integer arithmetic, identity is `===`, and the float / tolerance grab-bag is down to two named constants at the boundaries.

## Features

- **Smart stock suggester** (#24) — A banner above the colour-mapping panel clusters BOM parts by cross-section, matches against the timber-preset catalogue (AU DAR, AU posts, EU C24, pine glue panel, US Pine 1×10, etc.), and adds the lot with one click. Each suggested entry carries a per-cluster `oversize.crossSection` allowance plus a unit-default length allowance.
- **Material allowance on linear stock** (#24) — Collapsible "Material allowance" section on linear stock cards. The allowance flows through the matcher (`effectiveOversize`) and packer (`makePartRect`), and is annotated on the Layout view and PDF so the woodworker knows how much to plane down.
- **Universal model imports via Assimp WASM** (#25) — `.dae`, `.fbx`, and binary `.glb` files now convert to glTF JSON in-browser via assimpjs (~4 MB, lazy-loaded on first non-glTF drop) and flow through the existing glTF pipeline. Native `.gltf` keeps its fast path. SketchUp import guide gains the FBX "Units: Meters" and DAE "Triangulate all faces" gotchas.
- **Stock help in the left sidebar** — Persistent explanatory panel covering stock + material allowance, condensed for the narrow `ViewerSidePanel`.
- **Workspace tabs reordered** — BOM → Stock → Layout → Model → Build → Settings.
- **Empty stock list is an empty state** — Layout tab handles "no stock yet" as a guided empty state instead of an error.

## Bug Fixes

- **BOM showed L / W swapped on rotated placements** — Sheet placement serializer was overwriting `widthM` / `lengthM` with the packer's rotated axes, so two instances of the same part rendered with flipped dims when one was rotated. Intrinsic dims now stay stable across instances; diagram consumers read the on-board footprint from placement corners directly.
- **Identical parts split into separate BOM rows** — Sub-mm OBB drift between mesh instances (FBX exports differing by ~0.001 mm between identical boards) was finer than the 0.1 mm grouping tolerance, so visually-identical parts landed in different rows. Grouping tolerance raised to 1 mm, and OBB defers to the stable AABB when the two agree within 0.1 mm.
- **Inch ↔ mm FP slop end-to-end** — Preset boundary snaps to the 1 µm grid in mm representation (`toCanonicalMm`), so `3.5″` becomes exactly `88.9 mm`, not `88.89999…`. Downstream cluster gaps, the nonnegative-oversize Zod schema, and the UI all see clean values.
- **Stock rename / delete left dangling colour mappings** — Renaming or removing a stock now cascades into `colorMap` references on every model in the project, so the BOM doesn't keep pointing at a stock that's gone.
- **Vue reactivity leaked into IDB patches** — `useProjectSettings` was passing reactive proxies into Dexie writes; patches now strip reactivity before they hit the store.
- **Part / stock identity rejected drift-affected thicknesses** — OBB drift left cluster leaders a few µm below the nominal YAML stock value, so exact-equality matching falsely rejected "this 18 mm part fits this 18 mm stock." Restored a single named `STOCK_MATCH_TOLERANCE_UM = 500` applied only to part ↔ stock identity. Stock ↔ stock and placement geometry remain exact integer.

## Performance

- **Integer-µm placement geometry** (#26) — Free-rect splits, overlap tests, score arithmetic and rotation now operate end-to-end in integer micrometres. Identity is `===`; no more relative-magnitude epsilon walk through the packer.

## Breaking Changes

- **Schema v5 — internal dimensions are integer micrometres** — `IdbProject.{bladeWidth, margin}` and every `Part.size.*` are now branded `Micrometres`. Existing projects migrate automatically.
- **Schema v6 — stocks stored as `StockMatrix[]`** — `IdbProject.stock` was a YAML string parsed on every read and dumped on every write; every consumer wanted the structured array. Lifted to `stocks: StockMatrix[]` directly. Existing projects migrate automatically.
- **Model source label renamed `'collada'` → `'assimp'`** — Since the same import path now handles DAE, FBX, and GLB, the label reflects the converter, not the format. Split out of the v5 migration into its own pre-v5 step.

## Code Quality

- **Micrometres brand end-to-end** — `Micrometres = number & { __um: unique symbol }` prevents raw meters / mm from leaking into engine territory; erased at runtime so the worker boundary is a no-op. Engine fields mass-renamed `*M` → `*Um`. `floating-point-utils.ts`, `Config.placementEpsilon`, `isNearlyEqual` deleted — tolerance collapses to `GROUP_TOLERANCE_UM` (cluster step) and `STOCK_MATCH_TOLERANCE_UM` (part ↔ stock identity) with clear reasons.
- **Tolerance-cluster grouping over grid-snap rounding** — `groupPartInfos` clusters raw extents within a 1 mm window instead of rounding to a grid, so near-identical exports group cleanly without losing real distinctions.
- **Stock writes centralised in `useStockMutations`** — Add / update / remove all flow through the composable, so `StockTab` and the suggester banner are pure UI.
- **Packer interface cleanup** — Bin-state methods are now required; dead `addToPack` and unused geometry helpers dropped. `Rectangle.center` removed (zero callers, brand-laundered sub-µm values).
- **Viewer rendering scale fix** — `useGetPx` switched from `PX_PER_M` to `PX_PER_UM = 1 / 2000`. Pre-fix callers were feeding µm into a px-per-meter scale, producing positions ~1 × 10⁶× too large in some codepaths. `RulerMeasurement` / `SnapEdge` field-name lies (`positionM`, `offsetUm` holding the wrong unit) renamed and retyped.
- **`Config.{bladeWidth, margin}` branded via Zod**; `ONE_INCH_UM` constant added.
- **Test re-baseline** — 1081 tests passing across packer / engine / glTF / migration / component fixtures. Shared `web/test-utils/units.ts` for future tests. Tautological grouping and template-forwarding tests dropped.
