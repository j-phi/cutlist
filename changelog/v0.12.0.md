# v0.12.0 — 2026-05-11

_Since [v0.11.0] (2026-05-11)._

A routing and surface-area release. Cutlist gains a public Plans marketplace, a dedicated Projects page (replacing the History dropdown / archive concept), YouTube embeds in the Build doc, and a mobile-friendly pass across the project tabs. Under the hood, the URL is now the single source of truth for project + tab state, project tabs are real nested routes (so Three.js / Tiptap code-split per tab), and the workspace lives at `/build/:projectId/:tab`.

## Features

- **Plans marketplace** (#19) — New `/plans` directory of static `.cutlist` projects, backed by files in `web/public/plans/`. A predev/prebuild script validates each export, extracts assets + scene thumbnails to `/_build/<slug>/`, rewrites build-doc embed refs, and emits per-plan manifests plus a top-level index. The detail route renders the build doc through the same `.tiptap-doc` surface as the editor, so the read-only view matches the authoring view 1:1. Tiptap only loads on the detail route; no client-side gzip decode.
- **YouTube embeds in Build doc** (#20) — `@tiptap/extension-youtube` wired in with a NodeView wrapper so YouTube blocks get the same drag-handle and selection chrome as Image and 3D scene blocks. Marketplace render path uses the extension's stock iframe styled to 16:9.
- **Projects page** (#21) — New `/projects` page lists every project in the database (sorted by `updatedAt`, thumbnails from the first captured scene) and owns deletion plus a subtle Reset Database link. The History dropdown is gone — whether a project is open as a tab is now session UI state in localStorage, not a field on the project record.
- **Mobile UX pass across BOM / Layout / Model** (#22) — BOM table owns its own horizontal scroll on narrow screens, columns condense, and the inline preview hides on mobile. Layout tab top controls share one flex row so the stock filter wraps below the settings toolbar instead of overlapping. Model viewer single-finger touch routes only to OrbitControls (no phantom marquee); tap-to-select still works. AnnotationToolbar shows icons only on mobile and gains Hide / Reset actions for the current selection.
- **Top-bar nav** — Home / Plans / Projects icons in the top bar, sharing the same teal gradient active state as project tabs.

## Performance

- **Tab-level code splitting** — Promoting project tabs to nested routes means Three.js no longer ships in the BOM chunk — it only downloads when the user navigates to `/build/<id>/model`. Tiptap is similarly deferred to `/instructions`.
- **Projects-page IDB load** — Starts from scenes-with-thumbnails and narrows models via `anyOf` rather than deserialising every model's `rawSource` to render the thumbnail grid. `appendProject` drops the heavy `getProjectWithModels` round-trip in favour of a list refresh.

## Bug Fixes

- **Read-only Build view leaked authoring chrome** — Drag handles, scene-picker dropdown, image upload affordances, and empty caption inputs were rendering in the non-editable view. `BuildDocEditor` now provides an `editable` ref the embed NodeViews and `EmbedCaption` consume, so each block hides its authoring controls in view mode.
- **Plans link active-class on detail routes** — `/plans/<slug>` is now a nested child of a `pages/plans.vue` parent route, so Vue Router's matched-records check lights up the Plans top-bar link on both the index and detail routes without manual path inspection.
- **Single-finger touch fired phantom marquees** — Touch input in the Model viewer now routes only to OrbitControls; marquee selection stays pointer/mouse-only.
- **Layout filter overlapped settings on narrow screens** — Top controls share one flex row so the stock filter wraps cleanly.

## Breaking Changes

- **Workspace moved to `/build/:projectId/:tab`** — The root path is freed up for top-level pages (`/`, `/plans`, `/projects`). Existing bookmarks to `/<projectId>` or `/<projectId>/<tab>` no longer resolve.
- **Schema v4 — drop `archivedAt`** — Whether a project is open as a tab is session/UI state, not data about the project, so it moves to localStorage and the IDB record drops `archivedAt`. Existing data migrates automatically.

## Code Quality

- **Route is the single source of truth** — `useUrlSync` is gone. `activeId` and `tab` are computed reads from `useRoute().params`; all mutations go through `navigateTo`. Kills a class of state↔URL race bugs that surfaced once `/plans` existed alongside `/build`.
- **Project tabs as nested routes** — `pages/build/[projectId]/[[tab]].vue` (one component switching between six tab bodies) becomes a parent route holding the workspace shell plus six thin child routes. Tab nav is now `<NuxtLink :to="…" replace active-class="…">` — no `setTab` handler, no `useProjectTab`, no `v-if` ladder. Cmd+click and right-click on a tab work natively. Deletes `useProjectTab`, `ProjectWorkspace.vue`, and the now-obviated tests.
- **Editable-injection helpers** — Replace the exported symbol + manual `inject(KEY, ref(true))` with `provideEditable` / `useEditable` so the symbol stays module-private. Includes a read-only default so embed NodeViews rendered outside the editor (e.g. marketplace render path) don't crash.
- **Plans page extraction** — The ~150-line contribute section is now a self-contained `<ContributeCard/>`; the page itself is just the directory grid.
- **Hero backdrop unification** — `HeroBackdrop` gains a `position` prop (`'center' | 'top'`) so the teal spotlight fades down from the top on inner pages (Plans, Projects) while the home hero stays centered.
